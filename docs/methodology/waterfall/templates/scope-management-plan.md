# Scope Management Plan (Waterfall)

## Purpose
Define scope definition, validation, and control for waterfall delivery.

## Audience
Project Manager, Business Analyst, PMO.

## Inputs
- Project charter (`docs/methodology/waterfall/templates/project-charter.md`).
- WBS (`docs/methodology/waterfall/templates/wbs.yaml`).
- Shared scope plan (`docs/templates/shared/scope-management-plan.md`).

## 1. Scope Definition Process
Describe how requirements are baselined and approved.

## 2. WBS Maintenance
Define ownership and version control for the WBS and dictionary.

## 3. Scope Validation
Identify gate reviews and acceptance criteria.

## Completion Checklist
- [ ] Scope statement and WBS approved.
- [ ] Change control thresholds documented.
- [ ] Acceptance criteria defined by phase.

## Acceptance Criteria
- Scope plan supports gate approvals and baseline traceability.
- Scope changes follow approved governance.
